Part I: The Foundation

For LLM agents, rigorous upfront planning and deliberate architectural design are not merely best practices—they are prerequisites for success. This section covers how to create a stable and predictable environment for agents to operate effectively.

Specification as Code

Shift your mindset: user stories, acceptance criteria, and ADRs are no longer just documents. They become the high-level source code that directly programs the agent's execution plan, minimizing ambiguity and defining success.

Test-First Prompting

Adapt Test-Driven Development for agents. By providing a failing test case in the prompt, you give the agent a precise, executable definition of "done," which is far more effective than natural language alone.

Architectural Guardrails

Use Architectural Decision Records (ADRs) as persistent context. An ADR stating "We use PostgreSQL for financial data" acts as a powerful, long-term instruction that prevents the agent from making incorrect architectural choices.

Repository Architecture: Monorepo vs. Polyrepo

The structure of your codebase has a profound impact on an agent's ability to understand and modify it. For agentic development, the choice is clear.

Monorepo (Recommended)

Provides a single, unified context. This is ideal for machine cognition, allowing agents to reason about system-wide dependencies and execute complex refactors with a higher success rate.

Polyrepo

Creates information silos. An agent working in one repository is blind to dependencies in another, making cross-cutting changes difficult and error-prone without significant manual intervention.

Core Reliability Techniques

Context Provisioning (RAG)

Use Retrieval-Augmented Generation as the agent's long-term memory. Index your codebase into a vector database to provide the agent with highly relevant, specific code snippets as context for its tasks, dramatically improving accuracy.

Structured Outputs & Tool Calling

Enforce predictability by requiring agents to respond in a strict JSON schema. Use the native "tool calling" features of modern LLMs to make agent actions robust, auditable, and easy to integrate with your existing code.

Safety via Patch/Diff & Guardrails

Never allow an agent to write directly to the filesystem. Instead, have it output a `git diff` patch for review. Combine this with automated guardrails that block disallowed actions (e.g., editing critical config files) and run linters/formatters on the proposed changes.
